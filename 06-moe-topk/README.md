# MoE Top-K

В слоях **Mixture of Experts (MoE)** маршрутизатор для каждого токена выдаёт скор (логит) на каждого эксперта. Дальше обычно берут `softmax`, выбирают **Top-K** (на практике это не очень большое значение, довольно распространненым является значение 8) и перенормируют веса — как в [Mixtral](https://huggingface.co/docs/transformers/model_doc/mixtral#transformers.MixtralSparseMoeBlock). **В этой задаче нужен только шаг выбора экспертов**: по матрице значений в **fp16** находятся `K` наибольших элементов в строке (в случае равенства скоров - выбирается элемент с наимешьим индексом), в выход записываются их **индексы** и **сами эти значения** из входа.

## Спецификация

Для каждого токена `t` (строка матрицы логитов):

1. Среди всех экспертов `e ∈ [0, numExperts)` выбрать `topK` различных индексов с **наибольшими** значениями `logits[t][e]`. На каждом шаге берётся текущий максимум среди ещё не выбранных.
2. При **равных** значениях предпочитать эксперта с **меньшим** индексом.
3. Записать индексы в `outIdxs[t, :]` и соответствующие значения логитов в `topkWeights[t, :]`.

Формат данных — **row-major**, шаги (`stride`) заданы **в элементах** соответствующего типа (как в задаче про квантизацию).
Реализуйте функцию (объявление в `moe_topk.cuh`):

```cpp
void MoeTopK(size_t batchSize, size_t numExperts, size_t topK, const __half* logits,
             size_t inputStride, int32_t* outIdxs, size_t idxsStride,
             __half* topkWeights, size_t outStride);
```

Внутри допускается только запуск вашего CUDA-ядра (и вспомогательных `__device__` функций); отдельных аллокаций и явного `cudaDeviceSynchronize` не требуется. Гарантируется, что логиты не принимают значения inf, -inf, NaN, а также   `topK ≤ numExperts`; `numExperts > 0 && numExperts <= 256`.
