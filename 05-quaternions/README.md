# Quaternions

В этой задаче вам предлагается реализовать перемножение набора кватернионов.


### Quaternion

Кватернион - это обобщение комплексного числа. Если каждое комплексное число $a + b \textbf{i}$ кодируется парой вещественных чисел $(a, b)$, 
то каждый кватернион $a + b \textbf{i} + c \textbf{j} + d \textbf{k}$ можно представлять в виде четвёрки вещественных чисел $(a, b, c, d)$.

Сейчас нас интересует произведение кватернионов.
Если не вдаваясь в подробности, формула произведения выглядит следующим образом:

$$
(a_1 + b_1 \textbf{i} + c_1 \textbf{j} + d_1 \textbf{k}) (a_2 + b_2 \textbf{i} + c_2 \textbf{j} + d_2 \textbf{k}) =
$$

$$
(a_1 a_2 - b_1 b_2 - c_1 c_2 - d_1 d_2) +
$$

$$
(a_1 b_2 + b_1 a_2 + c_1 d_2 - d_1 c_2) \textbf{i} +
$$

$$
(a_1 c_2 - b_1 d_2 + c_1 a_2 + d_1 b_2) \textbf{j} +
$$

$$
(a_1 d_2 + b_1 c_2 - c_1 b_2 + d_1 a_2) \textbf{k}
$$

Для подробностей можно посмотреть
* https://en.wikipedia.org/wiki/Quaternion#Multiplication_of_basis_elements
* https://en.wikipedia.org/wiki/Quaternion#Hamilton_product.

**Замечание #1** Операция перемножения кватернионов некоммутативна.

**Замечание #2** Для удобства мы уже предоставили вам класс `Quaternion`, а также вспомогательный класс `QuaternionMultiplier`, в котором мы уже реализовали логику перемножения кватернионов.

### Задание

Вы должны реализовать *перемножение* набора кватернионов, реализовав одну функцию со следующей сигнатурой:

```cpp
void QuaternionsReduce(size_t rows, size_t cols,
                       const Quaternion* inp, size_t inp_stride,
                       Quaternion* out, cudaStream_t stream);
```

**Пояснения**
* `rows` обозначает число входных наборов кватернионов, которые надо перемножить
* `cols` обозначает количество элементов в каждом наборе
* входные данные лежат в виде матрицы, которая имеет логические размерности `rows x cols` и лежит в row-major формате
* выходные данные - вектор размерности `rows`
* `inp` и `inp_stride` задают расположение входной матрицы в памяти
* `out` задаёт расположение вектора с выходными данными
* `stream` используется для запуска кернелов в отдельном, недефолтном стриме

**Замечание #1** **Гарантируется**, что `cols` принимает одно из трёх значених 1024 / 2048 и 4096, поэтому свой код можно писать, исходя из этого. В частности, циклы загрузки данных могут иметь фиксированную длину и легко анроллиться (что может уменьшить боттлнеки и ускорить код)

**Замечание #2** Структура `Quaternion` содержит 16 байт и выровнена по 16 байт, поэтому дефолтная загрузка целой структуры целиком будет выливаться в одну инструкцию 128-битной загрузки

**Замечание #3** Поскольку операция перемножения кватернионов **ассоциативна, но некоммутативна**, обратите внимание на то, чтобы у вас сохранялся правильный порядок перемножения кватернионов; при этом не нужно обеспечивать это во всех тредах - достаточно гарантировать, чтобы порядок перемножения кватернионов сохранялся хотя бы в том треде, который запишет результат (например тред #0).

**Замечание #4** Для большего понимания постановки задачи можно смотреть в CPU-реализацию в тестах.

**Замечание #5** Внутри функции `QuaternionsReduce` ничего аллоцировать и копировать **не следует** (в том числе аллоцировать буферы для хранения промежуточных переменных)
