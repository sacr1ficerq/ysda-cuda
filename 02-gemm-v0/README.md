# GEMM v0

В этой задаче вам предстоит реализовать один из основных вычислительных примитивов современных LLM - матричное умножение в виде операции [GEMM (general matrix multiplication)](https://en.wikipedia.org/wiki/Basic_Linear_Algebra_Subprograms#Level_3).
Точнее, необходимо реализовать вычисление следующей формулы:
$$
D_{m \times n} = \alpha A_{m \times k} \cdot B_{k \times n} + \beta C_{m \times n}
$$
Матрицы $C$ и $D$ при этом могут совпадать в памяти, то есть сложение может происходить in-place.

Каждая матрица может располагаться в памяти как по строкам (row-major формат), так и по столбцам (col-major формат):
![matrix_formats](matrix_formats.png).

При этом соседние строки/столбцы могут находиться не вплотную другу к другу, а с небольшими промежутками, обусловленными необходимостью выравнивания или, например, фактом получения матрицы как подматрицы какой-то другой без явного копирования.
Гарантируется, что матрицы $C$ и $D$ имеют один и тот же формат хранения в памяти.

Непосредственно в данной задаче требуется реализовать самую простую версию GEMM без использования разделяемой памяти.
В дальнейшем в рамках курса мы будем пытаться ускорять наивное решение, написанное в рамках данной задачи.

Сами матрицы имеют тип данных fp16, который представлен в CUDA типом данных `__half`, для использования которого необходимо подключить заголовочный файл `cuda_fp16.h`.
